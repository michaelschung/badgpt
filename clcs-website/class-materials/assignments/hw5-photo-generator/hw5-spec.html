In this assignment, you will create a website where you can scroll through images and save some as your favorites. Completing this assignment requires a strong grasp of buttons localStorage, and HTML redirects.

Tips

This assignment is pretty complex, so these tips are going to be more helpful than before. Here's a list of the major milestones, and some tips to go along with each. Pay close attention to details, and make sure you complete them in order!

1) Previous and Next buttons cycle through the photos

This section is super similar to the "Movie Posters" exercise that we did in class, so go look at your code from that day (the day titled "debugging"). If your code from that day isn't good, take a look at mine.
  • Notice that the images are all named using the pattern img#.jpg (where # is a number between 1 and 42), so there's no need to make a list of image names like we did with the movie posters. All you need to keep track of is the number of the current image.

2) Save button makes new HTML node in fav-bar

The next task is to make the Save button work. When clicked, it should figure out which image is currently being displayed, grab the filename, make a new <img> tag with the same source, and add the new element to the fav-box.
  • Remember that CSS rules are automatically applied to all relevant elements, even ones that are added in via JS code.
  • Eventually, the new page will need to have access to all of the favorite photos. Thus, it will be best to store the filenames all together in a single array. Remember that you can use arr.push(someValue) to add a value to an existing array.

3) Done button redirects to new page

When the Done button is clicked, it should redirect us to a new HTML page.
  • Make sure that you store the crucial information to localStorage before redirecting the page.
  • Remember that all localStorage values must be stored as strings. Use JSON.stringify(someValue) to turn any value (even an array) into a string. When you want to revert that string back into its original form, use JSON.parse(strValue).

4) New page uses script to load images

When the new page loads, it must call a script to grab the stored filenames and load them into itself.
  • Add the script from inside the <body> of the document.
  • Remember that when you link a script from HTML, the script is automatically run in its entirety.

Other general tips

  • Code a little, test a lot. Test your code after every few lines of new code. Don't write a whole bunch of code and then try to test it all at the same time – that's a surefire way to get yourself lost.
  • Remember to use the console! It'll tell you when there are errors, and you can use it to print things.
  • Speaking of printing, use console.log() liberally. It's your friend and it'll save your life. If something isn't working, start printing things to make sure they're exactly what you think they are.
  • Be careful with pathnames. Think about which file using the pathname, and direct it accordingly. Remember that you can use ../ to get out of the current folder.
