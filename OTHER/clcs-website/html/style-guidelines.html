Style Guidelines

Comments

Your code must have reasonable commenting throughout. If you're not sure how much is "reasonable," err on the side of commenting too much. I'll let you know if you're overdoing it, and I'll never take off points for extra comments.

A comment in HTML looks like this:

<!-- This is an HTML comment -->

A comment in CSS looks like this:

/* This is a CSS comment */

A comment in JavaScript looks like either of the following:

// This is a JavaScript comment

/* This is also a JavaScript comment, and
   its advantage is that it can span
   multiple lines. Everything between the
   slash-star and the star-slash is part
   of the comment. */

The shortcut for any comment is "Cmd /" ("Ctrl /" for Windows). Notice that this will comment out the entire line that your cursor is resting on.

Efficiency

Your code should be as efficient as possible; avoid redundancies.

CSS:

Remember that child elements inherit styling characteristics from their parents, so don't give the children redundant characteristics.

Make sure your CSS rules are in order from least to most specific. That means tag selectors come first, followed by class selectors, and then id selectors.

JavaScript:

Use loops/methods to make your code as efficient and non-redundant as possible.

Naming

Use concise, informative names, and follow good naming conventions.

HTML/CSS: All lowercase, with words separated by hyphens.

<div class="example-name">
    Contents of div
</div>

.example-name {
    /* blah blah */
}

JavaScript: Same as the conventions we used in Java.

Variables/methods should be named using lowerCamelCase.
We won't be dealing with object classes, so don't worry about UpperCamelCase and all that stuff.

Spacing

Use spacing to clearly delineate hierarches, and surround operators (such as +, -, =, {, }, etc.) with spaces.

HTML:

<body>
    I'm inside the body tag, so I'm
    tabbed over.
</body>

CSS:

.classname {
    /* blah blah */
}

JavaScript:

for (var i = 0; i < 10; i++) {
    // do stuff
}
